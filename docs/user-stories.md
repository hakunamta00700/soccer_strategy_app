# 유저 스토리

## Epic 1: 전술 보드 기본 기능

### US-1.1: 선수 배치하기
**As a** 코치  
**I want to** 전술 보드에 선수를 배치할 수 있어야  
**So that** 팀 포메이션을 시각적으로 표현할 수 있다

**Acceptance Criteria:**
- [ ] 사이드바에서 선수 아이콘을 드래그하여 전술 보드에 배치할 수 있다
- [ ] 배치된 선수는 등번호, 이름, 포지션을 표시한다
- [x] 선수를 클릭하여 속성(등번호, 이름, 포지션, 색상)을 수정할 수 있다
- [x] 배치된 선수를 드래그하여 위치를 변경할 수 있다
- [ ] 선수를 선택하고 Delete 키를 눌러 삭제할 수 있다
- [ ] 선수를 우클릭하여 컨텍스트 메뉴에서 복사/삭제할 수 있다

**Priority:** High  
**Story Points:** 5

---

### US-1.2: 포메이션 템플릿 사용하기
**As a** 코치  
**I want to** 미리 정의된 포메이션 템플릿을 사용할 수 있어야  
**So that** 빠르게 기본 포메이션을 설정할 수 있다

**Acceptance Criteria:**
- [x] 포메이션 선택 메뉴에서 4-4-2, 4-3-3, 3-5-2 등 기본 포메이션을 선택할 수 있다
- [x] 포메이션 선택 시 선수들이 자동으로 배치된다
- [ ] 기존 선수가 있으면 덮어쓸지 추가할지 선택할 수 있다
- [x] 포메이션 템플릿은 홈/어웨이 팀 구분을 지원한다

**Priority:** Medium  
**Story Points:** 3

---

### US-1.3: 전술 도형 그리기
**As a** 코치  
**I want to** 화살표, 선, 영역 등 전술 도형을 그릴 수 있어야  
**So that** 공의 이동 경로와 전술 영역을 표시할 수 있다

**Acceptance Criteria:**
- [x] 도구 팔레트에서 화살표, 직선, 곡선, 사각형, 원 도형을 선택할 수 있다
- [x] 선택한 도형을 전술 보드에 그릴 수 있다
- [ ] 그린 도형을 선택하여 색상, 두께, 스타일을 변경할 수 있다
- [ ] 도형을 드래그하여 위치를 이동할 수 있다
- [ ] 도형의 크기를 조절할 수 있다 (핸들 드래그)
- [x] 도형을 삭제할 수 있다

**Priority:** High  
**Story Points:** 8

---

### US-1.4: 전술 보드 줌/팬
**As a** 코치  
**I want to** 전술 보드를 확대/축소하고 이동할 수 있어야  
**So that** 세밀한 작업과 전체적인 뷰를 모두 확인할 수 있다

**Acceptance Criteria:**
- [ ] 마우스 휠로 줌 인/아웃할 수 있다
- [ ] 줌 슬라이더로 줌 레벨을 조절할 수 있다
- [ ] 스페이스바 + 드래그로 전술 보드를 이동할 수 있다
- [ ] "전체 보기" 버튼으로 전체 전술 보드를 한 화면에 표시할 수 있다
- [ ] 최소/최대 줌 레벨이 제한되어 있다

**Priority:** Medium  
**Story Points:** 3

---

### US-1.5: 격자 표시
**As a** 코치  
**I want to** 전술 보드에 격자를 표시할 수 있어야  
**So that** 선수와 도형을 정확하게 정렬할 수 있다

**Acceptance Criteria:**
- [x] 설정에서 격자 표시를 켜고 끌 수 있다
- [ ] 격자 간격을 조절할 수 있다
- [x] 격자에 스냅 기능을 켜고 끌 수 있다
- [x] 스냅이 켜져 있으면 선수와 도형이 격자에 자동 정렬된다

## Epic 2: 애니메이션

### US-2.1: 애니메이션 키프레임 생성
**As a** 코치  
**I want to** 애니메이션의 키프레임을 생성할 수 있어야  
**So that** 선수의 움직임을 단계별로 정의할 수 있다

**Acceptance Criteria:**
- [ ] 애니메이션 모드로 전환할 수 있다
- [x] 타임라인에서 키프레임을 추가할 수 있다
- [x] 각 키프레임에서 선수 위치와 도형 상태를 설정할 수 있다
- [x] 키프레임을 선택하여 수정할 수 있다
- [x] 키프레임을 삭제할 수 있다
- [x] 키프레임 간 시간 간격을 조절할 수 있다

**Priority:** High  
**Story Points:** 8

### US-2.2: 애니메이션 재생
**As a** 코치  
**I want to** 생성한 애니메이션을 재생할 수 있어야  
**So that** 선수들에게 전술 움직임을 보여줄 수 있다

**Acceptance Criteria:**
- [x] 재생 버튼으로 애니메이션을 시작할 수 있다
- [x] 일시정지 버튼으로 애니메이션을 멈출 수 있다
- [x] 정지 버튼으로 애니메이션을 처음으로 되돌릴 수 있다
- [x] 재생 속도를 조절할 수 있다 (0.5x ~ 2x)
- [x] 재생 중 타임라인에서 현재 위치를 확인할 수 있다
- [ ] 이전/다음 프레임 버튼으로 프레임 단위 이동이 가능하다

**Priority:** High  
**Story Points:** 5

---

### US-2.3: 애니메이션 타임라인 편집
**As a** 코치  
**I want to** 타임라인에서 애니메이션을 편집할 수 있어야  
**So that** 애니메이션을 정확하게 조정할 수 있다

**Acceptance Criteria:**
- [ ] 타임라인에서 키프레임을 드래그하여 시간 위치를 변경할 수 있다
- [ ] 타임라인에서 키프레임을 더블클릭하여 해당 프레임으로 이동할 수 있다
- [ ] 타임라인 줌 인/아웃으로 세밀한 편집이 가능하다
- [ ] 키프레임 간 보간 방식을 선택할 수 있다 (선형, 이징)
- [ ] 타임라인에서 구간을 선택하여 반복 재생할 수 있다

**Priority:** Medium  
**Story Points:** 5

---

### US-2.4: 선수 이동 경로 표시
**As a** 코치  
**I want to** 선수의 이동 경로를 시각적으로 표시할 수 있어야  
**So that** 선수들의 움직임 패턴을 명확하게 볼 수 있다

**Acceptance Criteria:**
- [x] 애니메이션 모드에서 선수의 이동 경로가 점선으로 표시된다
- [x] 경로 표시를 켜고 끌 수 있다
- [ ] 경로에 화살표를 표시하여 이동 방향을 나타낼 수 있다
- [ ] 각 선수의 경로 색상을 다르게 설정할 수 있다

**Priority:** Medium  
**Story Points:** 3

## Epic 3: 세션 관리

### US-3.1: 세션 생성
**As a** 코치  
**I want to** 새로운 훈련 세션을 생성할 수 있어야  
**So that** 각 훈련 세션별로 전술을 관리할 수 있다

**Acceptance Criteria:**
- [x] "새 세션" 버튼으로 세션을 생성할 수 있다
- [x] 세션 이름과 설명을 입력할 수 있다
- [x] 세션 생성 날짜가 자동으로 기록된다
- [x] 생성된 세션은 세션 목록에 표시된다
- [x] 세션을 생성하면 빈 전술 보드가 열린다

**Priority:** High  
**Story Points:** 2

---

### US-3.2: 세션 목록 보기
**As a** 코치  
**I want to** 저장된 세션 목록을 볼 수 있어야  
**So that** 이전에 작업한 세션을 쉽게 찾을 수 있다

**Acceptance Criteria:**
- [x] 세션 목록 화면에서 모든 세션을 볼 수 있다
- [x] 세션은 이름, 생성 날짜, 수정 날짜로 정렬할 수 있다
- [x] 세션을 검색할 수 있다 (이름, 설명)
- [x] 각 세션의 썸네일을 볼 수 있다
- [x] 최근 작업한 세션을 상단에 표시할 수 있다

**Priority:** High  
**Story Points:** 3

---

### US-3.3: 세션 불러오기

**I want to** 저장된 세션을 불러올 수 있어야

**Acceptance Criteria:**

- [x] 불러온 세션의 모든 전술과 애니메이션이 복원된다
- [x] 현재 작업 중인 세션이 있으면 저장 확인 메시지가 표시된다
- [x] 최근 세션 목록에서 빠르게 불러올 수 있다

**Priority:** High  
**Story Points:** 2

---

## Epic 4: 데이터 관리

### US-4.1: 자동 저장
**As a** 코치  
**I want to** 작업 내용이 자동으로 저장되기를 원한다  
**So that** 실수로 데이터를 잃어버리지 않는다

**Acceptance Criteria:**
- [x] 작업 내용이 주기적으로 자동 저장된다 (예: 30초마다)
- [x] 저장 상태가 UI에 표시된다 (저장됨/저장 중/미저장)
- [x] 브라우저를 닫았다가 다시 열어도 마지막 상태가 복원된다
- [x] 저장 실패 시 사용자에게 알림이 표시된다

**Priority:** High  
**Story Points:** 5

---

### US-4.2: 수동 저장
**As a** 코치  
**I want to** 언제든지 수동으로 저장할 수 있어야  
**So that** 중요한 시점에 명시적으로 저장할 수 있다

**Acceptance Criteria:**
- [x] Ctrl+S (또는 Cmd+S) 단축키로 저장할 수 있다
- [x] 메뉴에서 "저장" 옵션을 선택할 수 있다
- [x] 저장 완료 시 피드백이 표시된다
- [x] 저장 중에는 추가 작업이 차단되지 않는다 (비동기 저장)

**Priority:** Medium  
**Story Points:** 2

---

### US-4.3: JSON 내보내기/가져오기
**As a** 코치  
**I want to** 전술 데이터를 JSON 파일로 내보내고 가져올 수 있어야  
**So that** 백업하고 다른 컴퓨터에서 사용할 수 있다

**Acceptance Criteria:**
- [ ] "내보내기" 메뉴에서 JSON 형식으로 내보내기를 선택할 수 있다
- [ ] 세션 단위 또는 전술 단위로 내보낼 수 있다
- [ ] 내보낸 JSON 파일을 가져올 수 있다
- [ ] 가져올 때 기존 데이터와 병합할지 덮어쓸지 선택할 수 있다
- [ ] 잘못된 형식의 JSON 파일에 대한 에러 처리가 있다

**Priority:** Medium  
**Story Points:** 3

---

### US-4.4: 이미지 내보내기
**As a** 코치  
**I want to** 전술 보드를 이미지로 저장할 수 있어야  
**So that** 프레젠테이션이나 문서에 사용할 수 있다

**Acceptance Criteria:**
- [ ] "이미지로 저장" 메뉴에서 PNG 또는 JPG 형식을 선택할 수 있다
- [ ] 현재 전술 보드 상태가 이미지로 저장된다
- [ ] 해상도를 선택할 수 있다 (기본, 고해상도)

- [ ] 저장된 이미지 파일이 다운로드된다

**Priority:** Medium  
**Story Points:** 4

---

### US-4.5: 데이터 백업 및 복원
